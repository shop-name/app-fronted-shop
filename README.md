# ECサイト フロントエンド実装

モダンなフロントエンド技術を使用したECサイトのUIです。

> **注**: このプロジェクトはフロントエンドのみの実装です。バックエンドAPIはモックデータを使用しています。

## 🚀 デモURL

[https://app-fronted-shop.vercel.app](https://app-fronted-shop.vercel.app)

## 📋 概要

このプロジェクトは、モダンなフロントエンド技術スタックを学習し、実践するために開発したECサイトのフロントエンド実装です。Nuxt 3とTypeScriptを使用し、型安全で保守性の高いコードを意識して開発しました。

## 🛠 技術スタック

- **Nuxt 3** - Vue.jsベースのフルスタックフレームワーク
- **Vue 3** - Composition APIによるモダンなコンポーネント設計
- **TypeScript** - 型安全な開発（strict mode）
- **Pinia** - Vue 3公式推奨の状態管理ライブラリ
- **Vuetify 3** - Material Designコンポーネントライブラリ
- **SCSS** - スタイル記述

## ✨ 実装機能

### 商品一覧ページ

- ✅ 商品一覧表示（全18商品）
- ✅ ページネーション（8商品/ページ）
- ✅ ローディング状態の表示
- ✅ エラーハンドリング
- ✅ レスポンシブデザイン

### カート機能

- ✅ 商品をカートに追加
- ✅ 数量の増減（在庫数の上限チェック付き）
- ✅ カート内商品の削除
- ✅ 小計・消費税・合計金額の自動計算
- ✅ localStorageによるカート情報の永続化
- ✅ 在庫切れ商品の表示制御

### チェックアウト機能

- ✅ 注文内容の確認画面
- ✅ 請求額の内訳表示（小計・配送料・消費税・合計）
- ✅ 注文確定処理
- ✅ 注文完了後のトップページ遷移と通知表示

### 共通機能

- ✅ レスポンシブデザイン（モバイル・タブレット・デスクトップ対応）
- ✅ 統一されたエラーメッセージ管理
- ✅ SEOメタ情報の設定
- ✅ アクセシビリティ対応（alt属性など）

## 💡 学んだこと・工夫した点

### 型安全性

- TypeScript strict modeを有効化し、型安全な実装を徹底
- `ActionResult`型を定義し、ストアのアクション結果を統一的に処理
- インターフェースによる明確な型定義

### 状態管理

- Piniaを使用した効率的なグローバル状態管理
- カートストアと商品ストアの責務分離
- localStorageとの連携による永続化

### エラーハンドリング

- 在庫チェックとバリデーションの実装
- ユーザーフレンドリーなエラーメッセージ
- エラーメッセージの定数化による一元管理

### UX/UI

- ローディング状態の適切な表示
- 空の状態（カートが空など）のハンドリング
- Snackbarによる非侵襲的な通知
- Material Designに基づく統一感のあるデザイン

### パフォーマンス

- `computed()`を活用した計算結果のキャッシュ
- ページネーションによる表示データの制限
- コンポーネントの適切な分割

## 📁 プロジェクト構成

```
src/
├── assets/          # 静的アセット
├── components/      # Vueコンポーネント
│   ├── layout/      # レイアウトコンポーネント（Header, Footer）
│   └── ProductCard.vue  # 商品カードコンポーネント
├── constants/       # 定数定義
│   └── messages.ts  # メッセージ定数
├── data/            # モックデータ
│   └── products.ts  # 商品データ
├── layouts/         # レイアウト定義
│   └── default.vue  # デフォルトレイアウト
├── pages/           # ページコンポーネント
│   ├── index.vue    # トップページ（商品一覧）
│   ├── cart.vue     # カートページ
│   └── checkout.vue # チェックアウトページ
├── stores/          # Piniaストア
│   ├── cartStore.ts     # カート状態管理
│   └── productStore.ts  # 商品状態管理
└── types/           # 型定義
    └── common.ts    # 共通型定義
```

## 🚀 セットアップ

### 必要な環境

- Node.js 18.x以上
- npm または yarn

### インストール

```bash
# 依存関係のインストール
npm install

# 開発サーバーの起動
npm run dev
```

ブラウザで `http://localhost:3000` にアクセス

### ビルド

```bash
# プロダクションビルド
npm run build

# プレビュー
npm run preview
```
